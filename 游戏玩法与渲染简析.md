# 游戏玩法与渲染简析

本文档基于仓库内 `assets/core` 等脚本与场景配置，对 **玩法规则** 与 **画面/动画表现** 做简要说明，便于阅读代码或二次扩展。

---

## 一、项目与技术栈

- **引擎**：Cocos Creator **3.8.7**（见 `package.json`）。
- **形态**：2D 休闲解谜——**多瓶子、分层彩色液体**，通过点击选择瓶子并**倒水**，使液体按规则合并与分类。
- **关卡数据**：运行时通过 `CwgState.getData()` 加载 `resources/data/level_{关卡索引}.json`（`FunlandInfo` 解析为 `LevelStruct`：`level: GlassInfo[]`）。

---

## 二、玩法简析

### 2.1 基本操作

1. **第一次点击**某个可用瓶子：瓶子被「拿起」（位置上移、阴影偏移），并播放拾取音效（`SoundComp`）。
2. **第二次点击另一个瓶子**：
   - 若满足**倒水条件**（见下），则从第一个瓶子向第二个瓶子执行一次（可能连续多层）倒水，并播放成套动画。
   - 若不满足倒水条件，则先放下上一个选中瓶，再拿起当前瓶。
3. **再次点击同一瓶子**：仅放下，取消选中。

不可操作的瓶子：**已密封**的瓶子（见 2.3）。`VwFunland.isValidSelection` 会排除 `glass.isSealed()` 为真的对象。

### 2.2 倒水规则（核心逻辑）

在 `VwFunland.handleWaterTransfer` / `canTransferWater` 中：

- 源瓶**不能为空**，且不能处于「全部仍为隐藏层」的状态（`isAllHide`）。
- 目标瓶**未满**（每瓶最多 **4** 层水）。
- 目标瓶**为空**，或**顶层水颜色与源瓶顶层水颜色相同**时，可以倒入。
- 满足条件时，会**循环**倒出多层：只要仍可倒，就持续 `pourOutWater` → `addIntoWater`，即一次操作可转移**连续同色顶层的多段**（与常见「水排序」一致）。

倒水前会先把逻辑状态算完，再隐藏真实瓶子节点，用池化的动画节点播放过程；结束后恢复显示（`playPourOutWater`）。

### 2.3 密封（满瓶同色）

`Glass.isSealed()`：当瓶内恰好 **4 层**且**四段颜色全相同**时视为密封，自动**加盖**（`capNode`），且之后不能再被选为操作对象。

密封达成时，`VwEffect.showFullSeled` 会播放**粒子特效**，颜色与当前水色一致。

### 2.4 隐藏水与问号

`GlassInfo.hideCnt` 表示从底部起若干层在显示上被处理为「隐藏」：表现为黑色 + 问号子节点（`Glass.setWater`）。  
当源瓶倒完后若进入 `isAllHide()` 等条件，会触发 `showHide` 逐层揭露真实颜色（含 tween 变色）。

### 2.5 广告瓶

`GlassInfo.ad` 为真时，点击会走 `handleAdGlass`（当前实现为日志 + `removeAd`），用于「看广告解锁」类扩展。

### 2.6 UI 与道具

- **关卡号**：`VwUi` 显示「第 N 关」，数据来自 `CwgState.info.level`。
- **撤销**：一次成功倒水后，若目标瓶**未**因本次操作变为密封，会把双方倒水前的 `colors` 快照存入 `undoStack`；密封成功则清空撤销栈（`VwFunland.playPourOutWater`）。`VwUi` 根据 `undoStack` 是否为空控制撤销按钮是否可点。
- **交换 / 加空瓶**：通过 `VwExchange`、`VwAddGlass` 等组件与按钮事件 `handleProp` 触发（具体展示逻辑在对应 prop 脚本与场景中配置）。

### 2.7 关卡切换与编辑器

- `VwPlay.switchLevel` 调用 `FunlandInfo.preLevel` / `nextLevel` 并 `restartLevel`。
- `openLevelEditor` 加载 `editor` 场景；关卡编辑说明见仓库根目录 `README.md`（列数、颜色数、生成关卡等）。

### 2.8 关于「通关」标志

`VwFunland` 中存在 `finished` 字段，用于在结束时屏蔽触摸；当前工程内**仅见重置为 `false`**，**未见**将其设为 `true` 的关卡胜利逻辑。若需要明确通关弹窗或跳转，需在现有状态上补充判定与赋值。

## 三、渲染与表现简析

整体为 **2D UI / Sprite** 体系：瓶子、水柱、水面、水流条、阴影、瓶盖等均为节点 + `Sprite`（及 `UITransform`）组合，而非 3D 流体模拟。

### 3.1 水的颜色

`CwgConstant.ts` 中 `WaterColor` 枚举与 `WaterColors` 表为每种颜色提供 **base**（水体）与 **surface**（水面高光）两套十六进制色值；倒水与流动表现里会分别应用到不同 Sprite。

### 3.2 单层水的节点结构

每个水层节点大致包含：

- **子节点 0**：水面（可接自定义材质，见 `WaterSurface`）。
- **子节点 1**：问号等装饰。
- 本体 `Sprite`：水体主色。

`Glass.showSurface` 只让**最顶层**水的水面子节点显示，下层水面的子节点关闭，避免多层波纹叠在一起。

### 3.3 水面波纹（材质动画）

`WaterSurface` 在 `reset()` 时取当前激活水面的 `Sprite` 的 **材质实例**，设置 `randomSeed`、`waveAmplitude`、`waveFrequency`、`waveSpeed`、`edgeDamping`、`time` 等属性（自定义 Shader/Effect 参数），在 `update` 里递增时间并**衰减振幅**，实现短暂波纹后停止，避免持续高开销。

### 3.4 拿起 / 放下

对 `glassNode`、`shadowNode` 使用 **tween** 改变位置；阴影通过 `UIOpacity` 按水位层数映射不同透明度（`updateShadowOpacity`），增强体积感。

### 3.5 倒出动画（`GlassPourOut`）

- 继承 `Glass`，额外挂 **Animation**，在剪辑关键帧上触发 `pourOutStart` / `pourOutFinish`（与 `pourOutWater.anim` 等资源配置对应）。
- 倒出时用一组 `flowingSprites` 显示**流出**的色带，颜色与当前层一致。
- 结束时对动画做 **Reverse 倒播** 等处理，把瓶身姿态「收回」，再通过回调衔接回收与显示逻辑。

### 3.6 装入动画（`GlassFlowing`）

- **流动条** `flowingNode`：`Sprite` 染色 + `UITransform.height` 的 tween，模拟水流下落高度变化。
- **水位上升**：对应水层 `UITransform.height` 从较小值 tween 到正常高度，与流动条缩短同步。
- 若最终密封，播放瓶盖下落的 tween（`playPutOnCapAnim`）。

### 3.7 粒子与对象池

- **满瓶特效**：`EfParticleSystem2D` 封装 `ParticleSystem2D.resetSystem` 与简单生命周期后回调，供 `CwgPools` 池化复用。
- **对象池**（`CwgPools`）：玻璃瓶实例、倒出动画节点、接水动画节点、满瓶特效节点均从池中取用与归还，减少频繁 `instantiate/destroy` 的开销。

### 3.8 方向与层级

倒水时根据源瓶与目标瓶的 **x** 关系设置倒出动画节点的 `scale.x` 为 `1` 或 `-1`，使倾倒方向朝向目标；接水动画节点的 **siblingIndex** 会与场景中的瓶子对齐，减少遮挡错误。

## 五、小结

- **玩法**：点击选择瓶子，在规则允许下将源瓶顶部连续可倒的水层倒入目标瓶；同色或空瓶接收；4 层同色密封后不可再动；支持隐藏水、广告瓶、撤销与部分道具扩展。
- **渲染**：以 **Sprite + 自定义水面材质 + Animation + tween + 2D 粒子** 完成液体与反馈，**非物理流体**；动画实例与特效通过 **对象池** 管理，适合移动端反复倒水场景。
