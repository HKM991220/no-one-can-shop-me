# 项目结构说明

本文档用于说明 `watercocos` 项目的目录结构、核心脚本职责与关键资源用途，便于后续维护与扩展。

## 一、目录结构（核心）

```text
watercocos
├─ assets
│  ├─ mian.scene                      # 运行主场景（当前主入口）
│  ├─ editor.scene                    # 关卡编辑器场景
│  ├─ core/                           # 核心玩法逻辑
│  │  ├─ glass-anims/                 # 倒水/接水/水面动画
│  │  ├─ prop-anims/                  # 道具表现逻辑
│  │  └─ other-anims/                 # 其它动画/特效封装
│  ├─ common/                         # 通用工具层
│  │  └─ ui/                          # UI 框架模块
│  ├─ editor/                         # 编辑器模式脚本
│  ├─ resources/                      # 运行时动态加载资源
│  │  ├─ prefab/                      # loading/game/道具预制体
│  │  ├─ data/                        # level_*.json 关卡数据
│  │  ├─ sounds/                      # 音效资源
│  │  └─ images|bg|粒子/              # 图片/背景/粒子资源
│  └─ anims/                          # 动画与 effect 材质
└─ settings/v2/packages/              # 工程与引擎配置
```

## 二、核心脚本职责

### 1) `assets/core`

- `VwPlay.ts`：游戏入口控制器，初始化状态、重开关卡、切换关卡/编辑器。
- `VwFunland.ts`：主玩法流程（点选瓶子、倒水、撤销、加空瓶、触摸事件）。
- `Glass.ts`：单瓶行为与表现（水层、拿起放下、封盖、隐藏层显示）。
- `VwUi.ts`：主界面控制（关卡文本、撤销按钮、道具按钮联动）。
- `SoundComp.ts`：音频播放封装（拿起音、倒水音）。
- `VwEffect.ts`：特效桥接（如满瓶封盖时粒子）。
- `FunlandInfo.ts`：关卡数据模型与关卡切换逻辑。
- `CwgState.ts`：运行时状态与关卡数据读取入口。
- `CwgPools.ts`：对象池管理（玻璃瓶、动画节点、特效节点复用）。
- `CwgConstant.ts`：常量定义（水色枚举与颜色表）。

#### `assets/core/glass-anims`

- `GlassPourOut.ts`：倒水瓶动画控制（倾倒时序、事件回调、回位）。
- `GlassFlowing.ts`：接水动画控制（水位上升、水流、封盖）。
- `WaterSurface.ts`：水面材质动画驱动（波纹参数更新）。

#### `assets/core/prop-anims`

- `VwExchange.ts`：换色道具交互（选择目标瓶并执行随机换色）。
- `VwAddGlass.ts`：加空瓶道具逻辑（计算可放位置并创建空瓶）。

#### `assets/core/other-anims`

- `EfParticleSystem2D.ts`：2D 粒子播放与生命周期回调封装。

### 2) `assets/common`

- `EventMng.ts`：全局事件中心（on/off/emit）。
- `ResLoader.ts`：Bundle/Json 资源加载封装。
- `Pool.ts`：通用对象池。
- `Toolkit.ts`：工具函数（如等待 tween、对象拷贝）。
- `LocalStorage.ts`：本地存储封装。
- `Rand.ts`：随机工具。

#### `assets/common/ui`

- `UIManager.ts`：UI 注册、打开、关闭、栈管理。
- `UIBase.ts`：UI 基类（生命周期 + 事件/资源模块入口）。
- `UIEventModule.ts`：UI 事件监听与自动清理。
- `UIResModule.ts`：UI 资源加载、缓存与释放。
- `UILaunchFlow.ts`：启动流程（loading -> 预加载 -> game -> 关闭 loading）。
- `UIFrameworkExample.ts`：示例脚本（事件 + 异步加载用法）。

### 3) `assets/editor`

- `VwEdPlay.ts`：编辑器模式入口控制。
- `VwEdFunland.ts`：编辑区玩法视图管理。
- `VwEdUi.ts`：编辑器 UI 与导出入口。
- `VwEdGlassSetting.ts`：单个杯子的参数设置。
- `UiEdColumn.ts`：单列管理（增删、重排）。
- `UiEdColumnCtrl.ts`：列控制器包装。
- `UiEdColumnCount.ts`：列数量管理。
- `UiEdColorCount.ts`：颜色数量与分配管理。
- `EdGlass.ts`：编辑器专用玻璃杯组件。
- `EdFunlandInfo.ts`：编辑器关卡数据模型。

## 三、关键资源用途

- `assets/resources/prefab/loading.prefab`：加载页预制体。
- `assets/resources/prefab/game.prefab`：游戏主预制体。
- `assets/resources/prefab/ExchangeItem.prefab`：换色道具目标提示。
- `assets/resources/prefab/AddEmptyGlass.prefab`：加空瓶道具表现。
- `assets/anims/water-wave.effect`：水面波纹 shader。
- `assets/anims/pourOutWater.anim`：倒水动画资源。
- `assets/resources/data/level_*.json`：关卡数据（位置、颜色、隐藏层、广告位等）。

## 四、运行链路（简版）

1. `UILaunchFlow` 打开 `loading` 预制体。
2. 预加载 `game.prefab` 及必要资源。
3. 打开 `game.prefab`。
4. `VwPlay` 初始化 `CwgState` 与 `FunlandInfo`，调用 `restartLevel()`。
5. `VwFunland` 创建瓶子并启动触摸交互；`VwUi` 同步按钮与关卡显示。
